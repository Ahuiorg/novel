## ADDED Requirements

### Requirement: 移动优先响应式断点系统
应用 SHALL 采用移动优先（Mobile-First）的断点策略，在以下标准断点上提供优化的布局体验：
- 基础（< 640px）：移动端布局
- sm（>= 640px）：平板竖屏、大屏手机横屏
- md（>= 768px）：平板横屏
- lg（>= 1024px）：笔记本、桌面端

#### Scenario: 移动端基础布局（< 640px）
- **WHEN** 用户在屏幕宽度小于 640px 的设备上访问应用
- **THEN** 时间线卡片宽度应使用 `min(320px, calc(100vw - 48px))` 自适应
- **AND** 底部导航采用紧凑布局，导航项间距减少
- **AND** 顶部导航 logo 字体缩小，操作按钮仅显示图标

#### Scenario: 平板布局（>= 640px）
- **WHEN** 用户在屏幕宽度大于等于 640px 的设备上访问应用
- **THEN** 时间线卡片恢复标准 320px 宽度
- **AND** 导航栏显示完整文字标签

#### Scenario: 桌面布局（>= 1024px）
- **WHEN** 用户在屏幕宽度大于等于 1024px 的设备上访问应用
- **THEN** 布局与当前桌面端设计保持一致

### Requirement: Fluid Typography 流式字体系统
应用 SHALL 使用 CSS `clamp()` 函数实现字体大小的平滑缩放，避免断点处的跳跃式变化。

#### Scenario: 标题字体平滑缩放
- **WHEN** 用户在不同宽度的设备上查看页面标题
- **THEN** 标题字体大小应在最小值和最大值之间平滑过渡
- **AND** 例如 `--text-3xl: clamp(1.5rem, 1.25rem + 1.25vw, 2rem)`

#### Scenario: 正文字体保持可读性
- **WHEN** 用户在移动设备上阅读正文内容
- **THEN** 正文字体不应小于 14px（0.875rem）
- **AND** 字体大小应根据视口宽度适当调整

### Requirement: 动态视口单位支持
应用 SHALL 使用动态视口单位（dvh, dvw）替代传统视口单位（vh, vw），以正确处理移动端浏览器 UI 的动态变化。

#### Scenario: 全屏高度在移动端正确显示
- **WHEN** 用户在移动端浏览器访问应用
- **THEN** 全屏元素应使用 `100dvh` 而非 `100vh`
- **AND** 当浏览器地址栏隐藏/显示时，布局应正确适应
- **AND** 底部固定导航不应被浏览器 UI 遮挡

### Requirement: 触屏交互优化
应用 SHALL 确保所有交互元素在触屏设备上具有足够的点击区域，遵循 Apple Human Interface Guidelines 的 44x44pt 最小尺寸要求。

#### Scenario: 导航按钮触屏友好
- **WHEN** 用户在触屏设备上点击导航按钮
- **THEN** 按钮最小点击区域应为 44x44 像素
- **AND** 按钮之间应有足够间距（至少 8px）避免误触

#### Scenario: 时间线卡片触摸交互
- **WHEN** 用户在触屏设备上滑动时间线
- **THEN** 滑动应流畅响应触摸手势
- **AND** 点击卡片应有明确的触摸反馈

### Requirement: 模态框响应式布局
应用中的模态框（搜索、设置、分享等）SHALL 在移动端提供适配的布局。

#### Scenario: 搜索模态框移动端布局
- **WHEN** 用户在屏幕宽度小于 640px 的设备上打开搜索模态框
- **THEN** 模态框宽度应为 `calc(100vw - 32px)`
- **AND** 模态框顶部距离应减少（约 5vh）
- **AND** 搜索类型按钮应支持水平滚动或自动换行

#### Scenario: 模态框内容滚动
- **WHEN** 模态框内容超出可视区域
- **THEN** 内容区域应支持独立滚动
- **AND** 模态框头部和操作按钮应保持可见

### Requirement: 响应式操作提示
应用 SHALL 根据设备类型显示适当的操作提示。

#### Scenario: 移动端显示触摸提示
- **WHEN** 用户在触屏设备上访问首页时间线
- **THEN** 操作提示应显示「← → 滑动浏览 · 点击卡片查看详情」
- **AND** 不应显示键盘快捷键提示

#### Scenario: 桌面端显示键盘提示
- **WHEN** 用户在非触屏设备上访问首页时间线
- **THEN** 操作提示应显示「← → 滚动浏览 · 点击卡片查看详情」
